# 采购需求调研

## 功能说明

根据《中国人寿再保险有限责任公司采购管理办法》（中再寿险发〔2026〕21号，以下简称"办法"）要求，通过结构化问答方式全面收集采购项目信息。采用动态问题策略，根据用户回答智能调整后续问题，确保信息收集完整性和交互体验。调研结果直接支撑后续 `procurement-overview` 任务概述生成及材料撰写。

## 触发场景

- 开始采购工作前的信息收集
- 需要撰写采购立项申请材料
- 用户询问"采购需求调研"、"收集采购信息"、"开始采购流程"等

## 调研流程

### 阶段零：预算与计划（必问）

在收集项目具体信息前，先确认采购项目的预算与计划状态，这是整个采购流程的前置条件。

**依据办法第8条（预算管理）、第9条（采购计划）：**

1. **预算状态**：该项目是否已纳入年度费用及资本性支出预算？
   - 已纳入年度预算，批复合规
   - 已纳入季度预算
   - 尚未纳入预算（→ 提示：依据办法第8条，需求部门应在每年编制次年年度预算时统筹考虑采购项目预算；未列入预算的项目需先完成预算审批程序）
   - 由财务管理部门在年度预算内调剂使用

2. **采购计划状态**：该项目是否已纳入年度集中采购计划或分散采购计划？
   - 已纳入年度集中采购计划（集采委已审批）
   - 已纳入年度分散采购计划（财务管理部门已备案）
   - 属于计划外项目（→ 提示：依据办法第9条，计划外集中采购项目需经集采委审批后方可执行）

### 阶段一：核心信息（必问）

**第1组：项目基础（依据办法第29条、第5条）**

3. **项目名称**：请提供采购项目的正式名称
4. **需求单位**：公司总部 / 分公司（若为分公司，需明确具体分公司名称）
5. **需求部门**：提出采购需求的具体部门
   - 若存在归口管理部门，后续单独收集（见第8问）
6. **采购类别**：工程类 / 货物类 / 服务类（依据办法第5条）
   - 提示：日常业务招待、食品、自办会务和临时所需的小额购买不属于本办法管理的采购行为
7. **预算金额**：预计采购金额（万元）（依据办法第29条）
8. **金额类型**（依据办法第53条，确定单次采购金额的规则）：
   - **固定金额**：以该固定金额确定单次采购金额
   - **有明确上限**（框架协议等）：以该上限确定单次采购金额
   - **预算金额**（无固定金额和上限时）：以该采购行为的预算金额确定
   - **预估发生额**（以上方法均无法适用时）：以经批准的预估发生额确定

**第2组：组织信息（依据办法第12-16条）**

9. **归口管理部门**：如有，请提供（依据办法第12条、第15条）
   - 归口管理部门参与集采委审批环节（其分管/协管领导及负责人将作为集采委成员）
10. **项目联系人**：负责本项目的联系人姓名及联系方式（依据办法第9条）

### 阶段二：采购特征（动态问题，依据预算金额分叉）

**核心分叉逻辑（依据办法第3条、第25条、第26条）：**

> **单次采购金额 ≥ 50万元** → 进入集中采购通道（A路径）
> **20万元 ≤ 单次采购金额 < 50万元** → 进入分散采购通道（B路径）
> **单次采购金额 < 20万元** → 进入自行采购通道（C路径）

---

#### A路径：集中采购（预算 ≥ 50万元）

11. **是否属于集中采购目录？**（依据办法第3条、附件1）
    - 属于集中采购目录 → 按集中采购管理
    - 不属于集中采购目录，但金额 ≥ 50万元 → 仍按集中采购管理（依据办法第25、26条反向推论）

12. **是否属于"三重一大"决策范围？**（依据办法第37条(四)）
    - 对于属于党委会前置研究事项清单、"三重一大"决策范围的重大项目立项，需提交党委会研究讨论
    - 若不确定，提示用户咨询部门负责人或查阅公司"三重一大"事项清单

13. **是否超出董事会授权范围？**（依据办法第37条(四)）
    - 超出董事会授权范围内的采购事项，需根据公司章程及授权方案规定履行相应审批

14. **潜在供应商数量：**
    - ≥3家 / 2家 / 仅1家（唯一供应商）
    - 若为2家，需在后续采购方案中说明市场上仅有2家（依据办法第21、22条）

15. **技术复杂程度：** 标准化 / 复杂 / 特殊（依据办法第21、22条）
16. **时间紧迫性：** 紧急（→ 说明紧急原因）/ 常规（依据办法第20、21条）
17. **是否计划外集中采购项目？**（依据办法第9条）
    - 计划外项目需经集采委审批后方可执行

**特殊采购类型（A路径中条件触发）：**

18. **是否属于联合采购？**（依据办法第3条、第41条）
    - 集团公司统一组织，由集团公司相关部门按集团公司采购管理办法执行
    - 涉及公司审批的事项，按公司相关审批权限进行

19. **是否委托招标代理机构？**（依据办法第27条）
    - 委托招标代理机构执行招标类集中采购，严格落实"管采分离""采需分离"
    - 也可委托外部代理机构办理招标类集中采购以外的采购事宜

20. **是否属于IT类委托采购？**（依据办法第16条(五)、第42条）
    - 委托中再数科/共享信息技术服务的非保险子公司开展的IT类采购
    - 信息技术部门统筹组织受托方按公司采购管理办法履行内部审批程序

21. **采购完成时间要求：**（依据办法第9条，需在采购计划中明确）

---

#### B路径：分散采购（20万 ≤ 预算 < 50万元）

22. **是否属于集中采购目录？**
    - 若属于集中采购目录但金额不足50万元 → 特殊情形，需确认处理方式（依据办法第3条）

23. **潜在供应商数量：** ≥3家 / 2家 / 仅1家

24. **是否计划外分散采购？** → 分散采购计划由财务管理部门留存备案（依据办法第9条）

25. **采购方式偏好：** 可参照集中采购方式要求选择合适的采购方式（依据办法第25条）
    - 无偏好 → 后续由概述阶段推荐

26. **采购完成时间要求**

---

#### C路径：自行采购（预算 < 20万元）

27. **是否属于集中采购目录？**
    - 若属于集中采购目录 → 虽金额不足50万，但仍需关注目录约束
    - 若不属于 → 按办法第26条，质优价廉、便利高效原则自行采购

28. **采购完成时间要求**

---

### 阶段三：安全保密（条件触发）

**触发条件判断逻辑：**

29. **是否涉及国家秘密或商业秘密？**（依据办法第24条(四)、第35条）
    - 涉及国家秘密 → 需保密审批，可不进行信息公开（但须说明理由）
    - 涉及商业秘密 → 同上级处理

30. **是否涉及核心经营信息？**（依据办法第28条、第29条(七)）
    - 触发条件：采购类别为"服务类"且涉及中介服务
    - 若涉及 → 需提供中介机构信息安全风险评估报告及保密责任状

31. **是否涉及外部数据采购？**（依据办法第29条(一)、第33条(五)、第39条(三)）
    - 若涉及 → 需提供数据安全自评估报告
    - 审批时需报大数据服务部、信息技术部和法律合规部

### 阶段四：前期工作（依据办法第29条，必问）

32. **项目必要性论证：** 已完成 / 进行中 / 未开始 / 需协助
33. **项目可行性论证：** 已完成 / 进行中 / 未开始 / 需协助
34. **市场调研情况：** 已完成 / 进行中 / 未开始 / 需协助
35. **同业调研情况：** 已完成 / 进行中 / 未开始 / 需协助
36. **潜在供应商寻源情况：** 已掌握（≥3家候选）/ 部分掌握（1-2家）/ 待寻源

### 阶段五：供应商信息（依据办法第29条(六)、第44条，条件触发）

37. **是否有候选供应商？**（公开招标除外）
38. **候选供应商基本情况：**
    - 依法注册成立，具有独立运作能力
    - 符合ESG行为准则要求（环境管理、社会责任、企业管治）
    - 无公司不可接受的不良记录

39. **是否需要供应商考察/尽职调查？**

### 阶段六：补充信息（可选）

40. 是否已有合同草案或合同关键条款意向？（依据办法第31条，采购文件合同主要条款需经法律合规部门审核）
41. 其他特殊要求或需说明的情况

## 交互策略

### 问题优先级
| 优先级 | 内容 | 跳过策略 |
|--------|------|----------|
| **P0（必须）** | 项目名称、需求单位、需求部门、预算金额、金额类型、采购类别、预算状态 | 不允许跳过 |
| **P1（重要）** | 计划状态、归口管理部门、供应商数量、"三重一大"判定、前期工作状态 | 可暂记为"待确认" |
| **P2（条件）** | 安全保密（国家秘密/商业秘密/核心经营信息/外部数据）、联合采购、委托代理、IT类委托 | 条件触发，不触发不询问 |
| **P3（补充）** | 时间要求、候选供应商、项目联系人、合同意向 | 可跳过 |

### 动态调整规则
- 预算 < 20万 且不属于集中采购目录：跳过阶段二大部分问题，直接收集阶段四、六
- 20万 ≤ 预算 < 50万：B路径，收集按需调整
- 预算 ≥ 50万：A路径全量问题
- 预算 ≥ 100万：必须确认"三重一大"（阶段二第12问升级为P0）
- 涉及中介服务：阶段三全部问题升级为P1
- 涉及外部数据采购：必须询问数据安全评估状态
- 涉密项目：跳过信息公开相关问题，收集保密级别详细信息
- 联合采购：按集团公司采购管理办法执行，简化公司内部流程问题

### 交互规范
- 每轮提问1-2个问题，避免信息过载
- P0类信息收集后需用户二次确认
- 敏感信息类问题前置提示"以下问题涉及项目敏感信息，仅用于合规判断"
- 用户可随时查看已收集信息（"查看已收集信息"指令）
- 用户可随时修改已回答的问题（"修改第X问"指令）
- 支持"暂不确定"跳过非P0问题（后续在概述阶段标注待确认项）

## 注意事项

1. 调研结果将作为后续材料撰写的基础，请尽量提供准确信息
2. 涉及国家秘密/商业秘密的问题将严格保密处理
3. 关键信息（P0类）缺失可能影响后续采购方式推荐和材料完整性
4. 用户可随时查看已收集信息或修改之前的回答
5. 对于不确定的问题，用户可选择"暂不确定"（记录至 `pending_confirmations` 字段）
6. **计划外集中采购项目**需经集采委审批后方可执行（办法第9条）
7. **联合采购**由集团公司统一组织，涉及公司审批事项按公司权限进行（办法第41条）
8. **IT类委托采购**由信息技术部门统筹组织受托方履行公司内部审批程序（办法第42条）
9. 属于**党委会前置研究事项清单、"三重一大"**决策范围的重大项目立项，应提交党委会研究讨论（办法第37条(四)）
10. **未列入预算的采购项目**需先完成预算审批程序后方可推进（办法第8条）
11. 涉及**外部数据采购**的项目，需报大数据服务部、信息技术部、法律合规部审批（办法第39条(三)）
12. 涉及**核心经营信息的中介服务项目**，须提供中介机构信息安全风险评估报告及保密责任状（办法第29条(七)）
13. 单次采购金额的计算严格依据办法第53条规则确定
14. 日常业务招待、食品、自办会务和临时小额购买不属于本办法管理范围（办法第5条）
