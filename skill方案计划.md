# 采购材料编制Skill体系方案计划

## 一、总体目标

帮助用户根据《中国人寿再保险有限责任公司采购管理办法》（中再寿险发〔2026〕21号）的要求，高效完成采购材料的编制工作，覆盖从需求调研到材料撰写的全流程。

## 二、Skill体系架构

采用多Skill协作模式，各司其职：

```
用户请求
    │
    ├──────────────────┬──────────────────┐
    ▼                  ▼                  ▼
┌─────────────┐  ┌─────────────┐  ┌─────────────────┐
│ procurement │  │ procurement │  │ procurement     │
│ -requirements│  │ -overview   │  │ -document-writer│
│ (需求调研)   │  │ (任务概述)   │  │ (材料撰写)      │
└─────────────┘  └─────────────┘  └─────────────────┘
                                            │
                              ┌─────────────┴─────────────┐
                              ▼                           ▼
                    ┌─────────────────┐         ┌─────────────────┐
                    │ procurement     │         │ template-       │
                    │ -reviewer      │         │ manager         │
                    │ (材料审核)      │         │ (模板管理)       │
                    └─────────────────┘         └─────────────────┘
```

## 三、各Skill职责详情

### 3.1 procurement-requirements（需求调研Skill）

**职责**：
- 通过问答方式收集采购项目的基本信息
- 依据采购管理办法设置问题
- 确保信息收集完整，为后续材料撰写奠定基础

**核心问题设计**（按顺序）：

| 序号 | 问题 | 依据 | 目的 |
|------|------|------|------|
| 1 | 项目名称是什么？ | 办法第29条 | 明确采购标的 |
| 2 | 采购类别是什么？（工程/货物/服务） | 办法第5条 | 确定采购范围 |
| 3 | 采购预算金额是多少？ | 办法第8、29条 | 确定采购方式（集中/分散） |
| 4 | 是否有特殊采购方式要求？ | 办法第18-27条 | 确定适用采购方式 |
| 5 | 需求部门是哪个？ | 办法第15条 | 确定责任部门 |
| 6 | 是否涉及归口管理部门？ | 办法第15条 | 确定管理部门 |
| 7 | 是否有候选供应商？ | 办法第29条 | 供应商寻源情况 |
| 8 | 供应商数量？（≥3/2/1） | 办法第21-24条 | 确定采购方式细类 |
| 9 | 是否涉及国家秘密/商业秘密？ | 办法第24、35条 | 确定信息公开要求 |
| 10 | 是否涉及核心经营信息？ | 办法第28、33条 | 确定信息安全评估要求 |
| 11 | 是否涉及外部数据采购？ | 办法第33、39条 | 确定数据安全评估要求 |
| 12 | 采购完成时间要求？ | 办法第20条 | 确定时间紧迫度 |
| 13 | 采购需求描述？（详细说明采购内容） | 办法第29条 | 编制采购文件 |

**交互模式**：
- 每次询问1-2个问题，便于用户回答
- 对用户回答进行即时确认
- 支持用户跳过可选问题
- 最终输出结构化的需求信息

### 3.2 procurement-overview（任务概述Skill）

**职责**：
- 根据收集的需求信息，生成采购任务概述
- 明确采购方式、所需材料清单
- 供用户确认或修改

**输出格式**：

```markdown
# 采购任务概述

## 基本信息
| 项目 | 内容 |
|------|------|
| 项目名称 | XXX |
| 采购类别 | 工程/货物/服务 |
| 预算金额 | XX万元 |
| 采购方式 | 公开招标/邀请招标/竞争性谈判/竞争性磋商/询价/单一来源 |
| 需求部门 | XX部门 |
| 归口管理部门 | XX部门（如有） |

## 采购方式说明
- **分类**：集中采购 / 分散采购
- **适用情形**：（依据办法第XX条）
- **推荐理由**：（AI分析）

## 所需材料清单
| 序号 | 材料名称 | 依据 | 备注 |
|------|----------|------|------|
| 1 | 项目立项申请 | 办法第29条 | 需包含基本情况、可行性论证等 |
| 2 | 采购方案请示 | 办法第31条 | 包含采购需求、评审方法等 |
| 3 | 供应商资质材料 | 办法第44条 | 候选供应商基本信息 |
| 4 | ESG行为准则 | 办法第33条 | 供应商需签署 |
| 5 | 信息安全评估（如需） | 办法第33条 | 涉及核心经营信息时 |
| ... | ... | ... | ... |

## 下一步操作
- [ ] 确认本概述信息
- [ ] 开始撰写项目立项申请
- [ ] 开始撰写采购方案请示
```

### 3.3 procurement-document-writer（材料撰写Skill）

**职责**：
- 根据任务概述，**参考模板**撰写采购材料
- 支持分步骤撰写（提纲→章节→全文）
- 多种采购方式的材料模板

**撰写流程**：

```
用户选择要撰写的材料
    │
    ▼
查询并展示对应模板
    │
    ▼
用户确认使用模板
    │
    ▼
根据模板和用户需求生成材料提纲
    │
    ▼
用户确认/修改提纲
    │
    ▼
按章节逐个撰写（填充模板内容）
    │
    ▼
汇总全文
    │
    ▼
用户审核/修改
```

**支持的材料类型**：

| 材料名称 | 适用场景 | 依据 |
|----------|----------|------|
| 项目立项申请 | 所有采购项目 | 办法第29条 |
| 采购方案请示 | 所有采购项目 | 办法第31条 |
| 采购结果申请 | 采购结果确定后 | 办法第33条 |
| 单一来源采购公示 | 单一来源采购 | 办法第24条 |
| 供应商后评价表 | 合同执行后 | 办法第46条 |

**模板应用机制**：
1. 根据采购类别（工程/货物/服务）选择对应模板目录
2. 根据材料类型选择具体模板
3. 读取模板内容，根据用户需求填充变量
4. 生成个性化材料

### 3.4 procurement-reviewer（材料审核Skill）

**职责**：
- 审核采购材料是否完整、合规
- 对照采购管理办法检查要点
- 提出修改建议

**审核要点**：

| 审核项 | 依据 | 检查点 |
|--------|------|--------|
| 完整性 | 办法第29、33条 | 是否包含所有必需要素 |
| 金额准确性 | 办法第53条 | 单次采购金额计算是否正确 |
| 采购方式适用性 | 办法第18-27条 | 选择的采购方式是否合适 |
| 审批权限 | 办法第37-39条 | 是否对应正确的审批层级 |
| ESG要求 | 办法第44条 | 是否包含ESG行为准则 |
| 信息披露 | 办法第35条 | 是否需要公示 |

### 3.5 template-manager（模板管理Skill）

**职责**：
- 管理各类采购材料的模板
- 支持模板的查询、预览、应用
- 根据办法更新模板

**模板分类**：

```
templates/
├── 工程类/
│   ├── 项目立项申请模板.md
│   ├── 采购方案请示模板.md
│   └── 采购结果申请模板.md
├── 货物类/
│   ├── 项目立项申请模板.md
│   ├── 采购方案请示模板.md
│   └── 采购结果申请模板.md
├── 服务类/
│   ├── 项目立项申请模板.md
│   ├── 采购方案请示模板.md
│   └── 采购结果申请模板.md
└── 通用/
    ├── 单一来源采购公示模板.md
    ├── 供应商后评价表模板.md
    └── ESG行为准则确认函.md
```

## 四、采购方式与材料对照

| 采购方式 | 立项申请 | 方案请示 | 结果申请 | 特殊材料 |
|----------|----------|----------|----------|----------|
| 公开招标 | ✓ | ✓ | ✓ | 招标文件 |
| 邀请招标 | ✓ | ✓ | ✓ | 招标文件、邀请函 |
| 竞争性谈判 | ✓ | ✓ | ✓ | 谈判纪要 |
| 竞争性磋商 | ✓ | ✓ | ✓ | 磋商纪要 |
| 询价 | ✓ | ✓ | ✓ | 询价记录 |
| 单一来源 | ✓ | ✓ | ✓ | 公示+论证材料 |

## 五、附件处理

根据办法，涉及的附件包括：

| 附件 | 用途 | 处理方式 |
|------|------|----------|
| 附件1：集中采购目录 | 确定是否属于集中采购 | 自动对照判断 |
| 附件2：集中采购实施流程指引 | 流程参考 | 按需查阅 |
| 附件3：集中采购信息公开表 | 信息披露 | 自动生成 |
| 附件4：集中采购监督员管理规则 | 监督要求 | 按需查阅 |

## 六、实施步骤

### 第一阶段：基础建设
1. 创建 procurement-requirements 调研Skill
2. 创建 procurement-overview 概述Skill

### 第二阶段：材料撰写
3. 创建 template-manager 模板管理Skill
4. 建立模板库（各类型材料模板）
5. 创建 procurement-document-writer 撰写Skill（集成模板调用）

### 第三阶段：审核完善
6. 创建 procurement-reviewer 审核Skill
7. 完善各Skill的交互逻辑
8. 测试联调
