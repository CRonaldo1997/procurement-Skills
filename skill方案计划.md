帮助用户根据《中国人寿再保险有限责任公司采购管理办法》（中再寿险发〔2026〕21号）的要求，高效完成采购材料的编制工作，覆盖从需求调研到材料撰写的全流程。

## 三、各Skill职责详情

### 3.1 procurement-requirements（需求调研Skill）

**职责**：
- 通过问答方式收集采购项目的基本信息
- 依据采购管理办法设置问题
- 确保信息收集完整，为后续材料撰写奠定基础

**核心调研模块**（按顺序）：

| 模块 | 问题数 | 主要内容 | 依据 |
|------|--------|----------|------|
| 一、基本信息 | 6 | 项目名称、类别、预算金额、金额类型、集中采购目录、联合采购 | 第三条、第五条、第五十三条 |
| 二、组织信息 | 4 | 需求部门、归口管理部门、项目联系人、分管领导 | 第三十七条 |
| 三、预算与计划 | 7 | 年度/季度预算、计划外采购、立项时间、完成时点 | 第八条、第九条 |
| 四、IT采购特殊情形 | 1 | 是否属于委托集团下属共享信息技术服务的IT类采购 | 第四十二条 |
| 五、采购方式与供应商 | 4 | 采购方式偏好、供应商数量、招标代理机构委托 | 第十八条至第二十七条 |
| 六、中介服务专项 | 2 | 是否涉及中介服务、核心经营信息 | 第二十八条第二款、第二十九条第七款 |
| 七、安全保密 | 3 | 国家秘密、商业秘密、外部数据采购 | 第二十四条、第三十三条、第三十五条、第三十九条 |
| 八、时间与紧急程度 | 3 | 预计完成时间、紧急采购、不可预见紧急情况 | 第二十条、第二十一条、第二十四条 |
| 九、需求描述 | 3 | 采购需求详情、客观明确性、标的物特征 | 第二十二条、第二十三条 |
| 十、前期准备工作 | 6 | 必要性/可行性论证、市场/同业调研、供应商情况 | 第二十九条、第四十四条 |
| 十一、特殊情形 | 9 | 三重一大、党委会前置、董事会授权、节能环保、专利技术等 | 第三十七条、第六条、第二十四条 |

**交互模式**：
- 每次询问1-2个问题，避免信息过载
- 支持回答确认和修改
- 不确定的问题可选择"暂不确定"跳过
- 动态调整问题顺序，优先收集关键信息
- 对涉及保密信息的问题进行特别提示
- 根据已回答问题智能推断后续问题
- 对"是"的回答自动展开追问细节

**输出**：结构化的需求调研结果，包含11个模块的完整信息

### 3.2 procurement-overview（任务概述Skill）

**职责**：
- 根据收集的需求信息，生成采购任务概述
- 明确采购方式、审批权限、组织架构、材料清单
- 供用户确认或修改

**核心功能模块**：

| 模块 | 内容 | 依据 |
|------|------|------|
| 一、采购分类判断 | 集中采购/分散采购/IT委托采购/联合采购 | 第三条、第四十一条、第四十二条 |
| 二、单次采购金额确定 | 固定金额/上限金额/预算金额/预估发生额 | 第五十三条 |
| 三、采购方式推荐 | 招标方式（公开/邀请）、非招标方式（谈判/磋商/询价/单一来源） | 第十八条至第二十七条 |
| 四、审批权限分析 | 立项审批、方案审批、结果审批、特殊审批路径 | 第三十七条至第四十条 |
| 五、组织架构要求 | 各机构职责、项目组/评审组/验收组/监督员组建 | 第十一条至第十六条、第三十四条、第五十一条 |
| 六、供应商管理要求 | 准入条件、ESG要求、日常管理 | 第四十四条至第四十五条 |
| 七、材料清单生成 | 7个阶段的必需要件和按需要件 | 第八条、第九条、第二十九条、第三十一条至第三十五条、第四十六条至第四十八条 |
| 八、采购纪律与监督检查 | 回避制度、禁止行为清单 | 第四十九条至第五十条 |

**输出格式**：

```markdown
# 采购任务概述

## 基本信息
| 项目 | 内容 |
|------|------|
| 项目名称 | XXX |
| 采购类别 | 工程/货物/服务 |
| 预算金额 | XX万元 |
| 采购方式 | 公开招标/邀请招标/竞争性谈判/竞争性磋商/询价/单一来源 |
| 需求部门 | XX部门 |
| 归口管理部门 | XX部门（如有） |

## 采购方式说明
- **分类**：集中采购 / 分散采购
- **适用情形**：（依据办法第XX条）
- **推荐理由**：（AI分析）

## 所需材料清单
| 阶段 | 材料名称 | 类型 | 依据 |
|------|----------|------|------|
| 采购计划 | 采购计划表 | 必需 | 第八条、第九条 |
| 项目立项 | 项目立项申请、必要性/可行性论证、市场调研报告等 | 必需 | 第二十九条 |
| 采购实施 | 采购方案请示、采购文件、法律合规审核意见 | 必需 | 第三十一条、第三十二条 |
| 采购结果 | 采购结果申请、ESG行为准则 | 必需 | 第三十三条 |
| 合同验收 | 采购合同、验收报告 | 按需 | 第三十四条 |
| 信息公开 | 成交结果公告 | 必需（集中采购） | 第三十五条 |
| 后评价 | 供应商后评价表、使用情况后评价 | 按需（20万元以上） | 第四十六条 |

## 下一步操作
- [ ] 确认本概述信息
- [ ] 开始撰写项目立项申请
- [ ] 开始撰写采购方案请示
```

### 3.3 procurement-document-writer（材料撰写+模板管理Skill）

**职责**：
- 管理各类采购材料的模板（模板查询、预览、应用、更新）
- 根据任务概述，参考模板撰写采购材料
- 支持分步骤撰写（提纲→章节→全文）
- 多种采购方式的材料模板

**模板分类**：

```
templates/
├── 工程类/
│   ├── 项目立项申请模板.md
│   ├── 采购方案请示模板.md
│   └── 采购结果申请模板.md
├── 货物类/
│   ├── 项目立项申请模板.md
│   ├── 采购方案请示模板.md
│   └── 采购结果申请模板.md
├── 服务类/
│   ├── 项目立项申请模板.md
│   ├── 采购方案请示模板.md
│   └── 采购结果申请模板.md
└── 通用/
    ├── 单一来源采购公示模板.md
    ├── 供应商后评价表模板.md
    └── ESG行为准则确认函.md
```

**撰写流程**：

```
用户选择要撰写的材料
    │
    ▼
根据采购类别和材料类型选择对应模板
    │
    ▼
展示并预览模板内容
    │
    ▼
用户确认使用模板
    │
    ▼
根据模板和用户需求生成材料提纲
    │
    ▼
用户确认/修改提纲
    │
    ▼
按章节逐个撰写（填充模板内容）
    │
    ▼
汇总全文
    │
    ▼
用户审核/修改
```

**支持的材料类型**：

| 材料名称 | 适用场景 | 依据 |
|----------|----------|------|
| 项目立项申请 | 所有采购项目 | 办法第29条 |
| 采购方案请示 | 所有采购项目 | 办法第31条 |
| 采购结果申请 | 采购结果确定后 | 办法第33条 |
| 单一来源采购公示 | 单一来源采购 | 办法第24条 |
| 供应商后评价表 | 合同执行后 | 办法第46条 |

**模板应用机制**：
1. 根据采购类别（工程/货物/服务）选择对应模板目录
2. 根据材料类型选择具体模板
3. 读取模板内容，根据用户需求填充变量
4. 生成个性化材料
5. 支持模板的查询、预览、更新

### 3.4 procurement-reviewer（材料审核Skill）

**职责**：
- 审核采购材料是否完整、合规
- 对照采购管理办法检查要点
- 提出修改建议

**审核要点**：

| 审核项 | 依据 | 检查点 |
|--------|------|--------|
| 完整性 | 办法第29、33条 | 是否包含所有必需要素 |
| 金额准确性 | 办法第53条 | 单次采购金额计算是否正确 |
| 采购方式适用性 | 办法第18-27条 | 选择的采购方式是否合适 |
| 审批权限 | 办法第37-39条 | 是否对应正确的审批层级 |
| ESG要求 | 办法第44条 | 是否包含ESG行为准则 |
| 信息披露 | 办法第35条 | 是否需要公示 |
